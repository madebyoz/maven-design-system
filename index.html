Base Grid System
for Slides

Spatial design for composing beautiful slides on a fixed canvas. Not responsive like the web, but a system of composition closer to cinematic framing.

Overview

Grids for slide design are fundamentally different from those for web or apps. Slides compose information within a fixed 16:9 canvas (13.33" × 7.5"), so rather than responsive design, we must think in terms of Composition.

The Maven grid system is built on three layers: Margins (the sacred whitespace), Columns (the horizontal skeleton), and Zones (semantic vertical divisions). Together, they form the spatial grammar that enables AI to auto-generate beautiful slides.

Cinema × Slides: Both are about placing information within a fixed frame. Just as a film director decides framing, this grid system codifies the compositional power of slides.

Design Philosophy

This design system is built on three principles.

Semantic Clarity

Every grid element has a meaningful name. By defining tokens like margin.outer and zone.title instead of raw coordinates, AI can understand the structure of a slide.

Projection Resilience

Slides are not only viewed on screens but projected. The grid prevents text from being clipped at screen edges and elements from being too close to distinguish.

Content Adaptability

Whether a text-heavy slide or a data dashboard, both should be beautifully laid out on the same grid. The grid is both a rule and a flexible framework.

Canvas

The canvas is the origin of all measurements. The 16:9 aspect ratio is the standard for modern displays and projectors.

Gutter

The standard gutter width is 0.3 in. Wider than web ensures clear element distinction during projection.

Vertical Zones

Instead of a web-style baseline grid, the vertical axis is divided into semantic zones. Three distinct roles.

Vertical Zones
TITLE ZONE — ≈20%
CONTENT ZONE — ≈65%
FOOTER — ≈15%
ZoneTokenPositionRole
Titlezone.title0.5" – 1.7" from topSlide title & subtitle
Contentzone.content1.7" – 6.5"Body, charts, images, data
Footerzone.footer6.5" – 7.0"Page numbers, sources, logo

Flexible zone boundaries: These ratios are defaults. On a no-title slide, the Content Zone extends to the top; on dashboards, the Footer Zone may be omitted.

Base Unit

We adopt 0.25 in (18pt) as the minimum unit for aligning all sizes and positions.

Three reasons: high compatibility with PowerPoint's snap grid, 7.5" ÷ 0.25" = 30 rows of flexibility, and 13.33" ÷ 0.25" ≒ 53 columns for fine adjustments.

Design Tokens

By structuring the entire grid system as tokens, AI can understand spatial information and generate consistent layouts.

grid-tokens.yaml
grid:
  canvas:
    width: 13.333in
    height: 7.5in
    aspect: 16:9
    dpi: 96
  margin:
    top: 0.5in
    bottom: 0.6in
    left: 0.5in
    right: 0.5in
  columns: 6
  gutter: 0.3in
  zones:
    title:  { top: 0.5in, height: 1.2in }
    content: { top: 1.7in, height: 4.8in }
    footer: { top: 6.5in, height: 0.5in }
  base-unit: 0.25in
  spacing:
    xs: 0.125in    # 1/2 unit
    sm: 0.25in     # 1 unit
    md: 0.5in      # 2 units
    lg: 0.75in     # 3 units
    xl: 1.0in      # 4 units

6-col center — Quotes / messages

AI Transformation Pipeline

p0's AI receives user intent, passes through design tokens, and transforms into final coordinates. This three-stage pipeline enables intent-based slide generation.

Intent                 Pattern                   Coordinates
─────────────          ──────────────────         ─────────────────────
"Compare"         →    three-up               →   col:2+2+2, zone:content
"Make impact"     →    full-bleed             →   col:6, zone:all
"Show data"       →    four-up / dashboard    →   col:3+3, zone:content
"Key message"     →    text-image             →   col:4+2, zone:content
